# Regras de Desenvolvimento do Projeto

## Convenção de Commits
1. **Rastreabilidade de Requisitos**: Todo commit realizado no repositório deve obrigatoriamente especificar no corpo ou no título da mensagem quais requisitos funcionais (`RFxx`) ou não funcionais (`RNFxx`) foram implementados, modificados ou corrigidos naquela atualização.

   *Exemplo de mensagem de commit:*
   ```text
   feat: implementado timeout por inatividade (RNF76) e rascunho automático (RNF81)
   ```
